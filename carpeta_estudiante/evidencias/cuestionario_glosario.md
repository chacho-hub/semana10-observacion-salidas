## Glosario

| Término | Definición con tus palabras |
|---|---|
| Salida | Información que el programa muestra después de procesar datos. |
| Resultado esperado | El valor o comportamiento que se espera que produzca el sistema. |
| Resultado obtenido | El valor o comportamiento que realmente entrega el sistema. |
| Validación | Comprobar que los datos y resultados cumplen las reglas o el objetivo. |
| Mensaje de error | Aviso de que algo está mal y debe corregirse. |
| Mensaje de advertencia | Aviso de que hay un posible problema, pero no impide seguir. |
| Mensaje de éxito | Mensaje que indica que la acción se realizó correctamente. |
| Consola | Herramienta del navegador donde se muestran mensajes técnicos. |
| Inspector | Herramienta que permite ver el HTML, CSS y la ejecución de una página. |
| Live Server | Extensión que actualiza automáticamente la página mientras se edita. |
| Terminal | Ventana donde se escriben comandos y se ejecutan programas. |
| NaN | “Not a Number”, indica que un valor no es un número válido. |
| undefined | Valor que indica que una variable no tiene un valor asignado. |
| Debug | Proceso de buscar y corregir errores en el código. |
| Objetivo funcional | Lo que el sistema debe hacer desde el punto de vista del usuario. |

## Conclusión breve

Observar los resultados y validar las salidas es esencial para detectar cuando el sistema no cumple el objetivo. Si un mensaje es confuso o incorrecto, el usuario no sabrá qué corregir. Por eso es importante mostrar errores, advertencias y éxitos con claridad. La validación ayuda a garantizar que los datos ingresados sean válidos antes de seguir con el proceso. Esto mejora la confianza en la aplicación y reduce problemas futuros.
